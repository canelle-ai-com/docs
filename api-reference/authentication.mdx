title: "Authenticate with the Canelle AI API Using API Keys"
sidebarTitle: "Authentication"

---

All requests to the Canelle AI REST API must be authenticated with an API key. Your API key identifies your account and determines which projects you can access. There are no session tokens or OAuth flows — every request simply includes your key in the `Authorization` header.

## Getting an API Key

<Steps>
  <Step title="Sign in to your Canelle AI account">
    Go to [canelleai.com](https://canelleai.com) and sign in with your credentials.
  </Step>
  <Step title="Navigate to API Keys">
    Open **Settings → API Keys** from the main navigation menu.
  </Step>
  <Step title="Generate a new API key">
    Click **Generate new API key** to begin creating your key.
  </Step>
  <Step title="Enter a descriptive name">
    Give your key a name that reflects where it will be used — for example, `Production` or `Development`. This makes it easy to identify and revoke specific keys later.
  </Step>
  <Step title="Copy the key immediately">
    Your new API key is displayed only once. Copy it now and store it in a secure location such as a password manager or secrets vault. You will not be able to view it again after closing this screen.
  </Step>
</Steps>

## Using Your API Key

Pass your API key as a Bearer token in the `Authorization` header of every request:

```bash
curl https://api.canelleai.com/v1/conversations \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Security Best Practices

<Warning>
  Exposing your API key in client-side code or version control gives anyone who finds it full access to your Canelle AI account. Treat your API key like a password.
</Warning>

- **Never expose your API key in client-side JavaScript or public repositories.** Anyone who can read your code can steal your key and consume your credits.
- **Use environment variables to store keys.** Reference them as `CANELLE_API_KEY=your_key` in your environment rather than hardcoding them in source files.
- **Create separate keys for production and development.** This limits the blast radius if a development key is accidentally exposed.
- **Rotate keys regularly and revoke any that may have been exposed.** Revocation takes effect immediately, so an exposed key can be disabled without downtime if you have a replacement ready.

## Environment Variable Examples

Store your key in `CANELLE_API_KEY` and read it at runtime — never in your source code.

<CodeGroup>

```javascript Node.js
const apiKey = process.env.CANELLE_API_KEY;

const response = await fetch('https://api.canelleai.com/v1/chat', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ project_id: 'proj_xxx', message: 'Hello' })
});
```

```python Python
import os
import requests

api_key = os.environ['CANELLE_API_KEY']

response = requests.post(
    'https://api.canelleai.com/v1/chat',
    headers={'Authorization': f'Bearer {api_key}'},
    json={'project_id': 'proj_xxx', 'message': 'Hello'}
)
```

</CodeGroup>

## Revoking a Key

To revoke an API key, go to **Settings → API Keys** and click **Revoke** next to the key you want to disable. Revocation is immediate — any subsequent request that uses the revoked key receives a `401 Unauthorized` response.

<Note>
  If you lose a key, revoke it and generate a new one. Lost keys cannot be recovered.
</Note>
